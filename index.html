Eerste contact

GitHub, 18 februari 2014

"Het bouwen van een platform waarop we de groepsblogs konden posten" .... Het leek me wel een leuke uitdaging. De eerste belangrijke beslissing was een gepaste 'tool' kiezen om deze website mee te bouwen. Eerst werd er gekozen om met 'durpalgarden' te werken, doch al snel werd deze optie verworpen. Onze groep bestond uit acht leden en één van de beperkingen in durpalgarden is dat er maximum slechts vijf leden tezamen aan één website kunnen bouwen. Op dat moment vroeg ik me af: 'waarom dan niet github gebruiken?' Want alhoewel ik nog nooit eerder een website heb gemaakt, heeft dit mij altijd al geïnteresseerd.

De eerste stap leek me logisch, zoek informatie over het gebruik van Github. Helaas was dit niet zo simpel als ik eerst had gedacht. In alle 'tutorials' die ik vond, met betrekking tot het gebruik van 'Github', stond vermeld dat je het programma 'GIT' moet downloaden om, met behulp van de source coding in GIT, items toe te voegen of te wijzigen in Github. Ik ben geen informaticus, dus het werken met source coding was niet direct de oplossing voor mij. Een handleiding en een aantal handige websites (How the heck do I use GitHub, GitHub for beginners) met betrekking tot het gebruik van 'Git' leken me wel een oplossing te bieden. Na een paar uur sukkelen heb ik echter ook dat opgegeven. Na lang zoeken kwam dan toch een eerste lichtpunt naar boven, een website waar op een eenvoudige manier uitgelegd stond hoe je Github kunt gebruiken zonder behulp van source coding of het programma 'Git'. Wat je echter wel nodig hebt is een specifiek programma om documenten van je computer op de internet versie van GitHub te zetten.

De volgende stap was wat experimenteren en één en ander uit te testen. Een account had ik al aangemaakt. Dit was waarschijnlijk één van de, tot dan toe, makkelijkste stappen. Vervolgens heb ik een organisatie aangemaakt, 'encyfeestbende' genaamd. Daarna heb ik repositories toegevoegd. Zodoende heb ik mijn eerste stappen in het bouwen van een website gezet, maar daar zal ik in mijn volgende blogs verder op ingaan.
