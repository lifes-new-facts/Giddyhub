Eerste contact

GitHub, 18 februari 2014

"Het bouwen van een platform waarop we de groepsblogs konden posten" .... Het leek me wel een leuke uitdaging. De eerste belangrijke beslissing was een gepaste 'tool' kiezen om deze website mee te bouwen. Eerst werd er gekozen om met 'durpalgarden' te werken, doch al snel werd deze optie verworpen. Onze groep bestond uit acht leden en één van de beperkingen in durpalgarden is dat er maximum slechts vijf leden tezamen aan één website kunnen bouwen. Op dat moment vroeg ik me af: 'waarom dan niet github gebruiken?' Want alhoewel ik nog nooit eerder een website heb gemaakt, heeft dit mij altijd al geïnteresseerd.

De eerste stap leek me logisch, zoek informatie over het gebruik van Github. Helaas was dit niet zo simpel als ik eerst had gedacht. In alle 'tutorials' die ik vond, met betrekking tot het gebruik van 'Github', stond vermeld dat je het programma 'GIT' moet downloaden om, met behulp van de source coding in GIT, items toe te voegen of te wijzigen in Github. Ik ben geen informaticus, dus het werken met source coding was niet direct de oplossing voor mij. Een handleiding en een aantal handige websites (How the heck do I use GitHub, GitHub for beginners) met betrekking tot het gebruik van 'Git' leken me wel een oplossing te bieden. Na een paar uur sukkelen heb ik echter ook dat opgegeven. Na lang zoeken kwam dan toch een eerste lichtpunt naar boven, een website waar op een eenvoudige manier uitgelegd stond hoe je Github kunt gebruiken zonder behulp van source coding of het programma 'Git'. Wat je echter wel nodig hebt is een specifiek programma om documenten van je computer op de internet versie van GitHub te zetten.

De volgende stap was wat experimenteren en één en ander uit te testen. Een account had ik al aangemaakt. Dit was waarschijnlijk één van de, tot dan toe, makkelijkste stappen. Vervolgens heb ik een organisatie aangemaakt, 'encyfeestbende' genaamd. Daarna heb ik repositories toegevoegd. Zodoende heb ik mijn eerste stappen in het bouwen van een website gezet, maar daar zal ik in mijn volgende blogs verder op ingaan.

Organisaties maken

Giddy hub, 27 april 2014

Een handig stukje informatie dat ik gevonden hen, en dat wel handig bleek te zijn, was het maken van 'organisaties'. Maar wat is nu net een organisatie? Organisaties zorgen ervoor dat meerdere mensen tegelijkertijd aan dezelfde website bouwen. De naam die je aan jouw gecreëerde organisatie geeft, zal de naam worden van je website. Je kan dus zeggen dat deze organisatie het werk-platform zal vormen waar aan je aan jouw website kan werken.

Dit leek voor mij de geschikte manier om samen te bouwen aan onze groepswebsite. Na een organisatie genaamd 'encyfeestbende' aangemaakt te hebben, een naam die reeds vaststond als onze facebook werkgroep, heb ik vervolgens de andere leden van onze groep aan deze organisatie toegevoegd. De reden van deze blogs over GitHub is om anderen vertrouwd te laten worden met het werken met GitHub. Met dit doel in ogen houdend, volstaat deze korte uitleg zeker niet. Om deze simpele reden zal ik hier, gebruik makend van beeldmateriaal, met jullie de stappen overlopen tot het maken van een organisatie.

De eerste stap is logisch, log in op jouw account. Vervolgens kun je op twee manieren te werk gaan. De eerste manier is om op het 'plus teken', wat naast je accountnaam staat, in de rechterbovenhoek van je scherm te klikken. Je kan op dit moment kiezen om ofwel een nieuwe repositorie, ofwel een nieuwe organisatie te creëren. De repositories zullen besproken worden in een volgende blog. Momenteel kies je dus voor het aanmaken van een nieuwe organisatie.

De tweede manier bestaat eruit om op je accountnaam op de linkerkant van je scherm te klikken. Je kan dan wederom kiezen om een organisatie aan te maken. Heb je al een organisatie aangemaakt, dan kun je op deze manier de reeds bestaande organisatie(s) beheren.

vervolgens kun je jouw nieuwe organisatie een naam geven. Niet vergeten, je moet hiervoor wel een e-mailadres invoeren. Normaal wordt 'open source' gekozen als standaard plan. Ik neem aan dat, indien je een ander plan kiest, je ook meer mogelijkheden hebt om je website verder en beter uit te bouwen. Helaas moet je voor deze versies betalen, wat voor mij snel duidelijk aangaf dat dit niet het voor mij gewenste plan was. Eens je plan gekozen, klik je op de groen knop genaamd 'create organization'. Proficiat, je organisatie is nu officieel.

Nu kun je, als je wilt, andere leden toevoegen aan jouw organisatie. Heb je die (nog) niet? Geen zorgen. Klik op 'finish' en je kan verder van start. Je kunt andere leden nog altijd toevoegen op een ander moment.

Nu de organisatie gecreëerd is, kun je repositories creëren. Dit zal in een volgende blog uitgelegd worden.
